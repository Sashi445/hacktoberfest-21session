Creating an account on github.com

  • All you have to do is head on to github.com and sign up to create an account.

Installing Git

Install git on windows

Download windows installer for git from here.

Now open the command prompt or git bash (preferable) and run the following to check whether git is installed or not

git --version

Install git on linux

To install git on a linux machine from your shell run the following commands

sudo apt-get update
sudo apt-get install git

and run the following to check whether git is installed or not

git --version

Install git on Mac

Download the latest Git for Mac installer from here.

Follow the prompts to install Git.

Open a terminal and verify the installation was successful by typing:

git --version

If you have installed Homebrew to manage packages on OS X, you can follow these instructions to install Git: Open your terminal and install Git using Homebrew:

brew install git

Verify the installation was successful by typing

git --version

Configuring git in your local system

Same for all users

Windows users are suggested to use git bash for typing commands

git config --global user.name "your-username(same username from github)"
git config --global user.email "your-email(same email from github)"

Making your first pull request

Pre-requisites

  • You should have Git installed on your local machine.
  • You’ll also need to have or create a GitHub account.

Steps

Step -1 : Fork a repository

Choose a repository you want to contribute to and fork it

When you’re on the main page for the repository, you’ll see a Fork button on your upper right-hand side of the page, underneath your user icon:

Click on the fork button to start the forking process. Within your browser window, you’ll receive a notification that the repository you’re forking is being processed:

Step - 2 : Clone the repository

  • Copy the url from the code button in the repo.

  • Open your git bash in windows or terminal in mac or linux and type the following command.

git clone "The url your copied"
After running the command you should see something like this
  • Now cd into the cloned directory or repo and open it with your favorite editor
cd repositoryName
  • Now make a new branch by typing the following command
git checkout -b feature-branch
Once you enter the git checkout command, you will receive the following output:
Output:
Switched to branch 'feature-branch'
Step-3: Make your changes locally

You can make your changes from your code editor like nano, vim, vscode etc.

If your changes are simply appending a line to the file then you can use bash commands like echo

Step-4: Commit your changes

Once you have modified an existing file or added a new file to the project, you can add it to your local repository, which we can do with the git add command. This command uses the following

git add .

This command adds all your modified files to staging area.

git commit -m "Commit Message"

This command commits all your changes made so far in simple words it creates a checkpoint in your history.

Step-5: Push your changes

You can now navigate to the forked repository on your GitHub webpage and toggle to the branch you pushed to see the changes you have made in-browser.
